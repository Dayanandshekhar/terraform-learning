Terraform AWS Provider - Contributor Guide

The AWS Provider is the work of thousands of contributors, and is maintained by a small team within HashiCorp. There are many way to contribute, and we have extensive documentation on them. Below are the main ways most users engage with the provider repository and the high level steps required to ensure the contribution goes smoothly.

What would you like do do?

Contribution Type Description Steps
Add a new AWS Service to the Provider Alow Terraform (via the AWS Provider) to manage an entirely new AWS service by introducing the resources and datasources required to manage configuration of the service.
  • Add Service Client
  • Perform Service Design
  • Add a new resource
Add a new resource to the Provider Allow the management of a logical resource within AWS by adding a new resource to the Terraform AWS Provider..
  • Name the resource
  • Fork the Provider
  • Fill out the Resource Schema
  • Implement CRUD Handlers
  • Write passing Acceptance Tests CRUD Handlers
  • Create Documentation for the Resource
  • Ensure format and lint checks are passing
  • Raise a Pull Request against hashicorp/terraform-provider-aws
  • Wait for Prioritization
